# Frontend Design Skill

Source: https://github.com/anthropics/skills/tree/main/skills/frontend-design

## Overview

This skill creates distinctive, production-grade frontend interfaces that prioritize high design quality and avoid generic AI aesthetics.

## When to Use

Apply this skill when the user requests web components, pages, applications, or any web UI work. Before writing any code, establish a bold aesthetic direction.

## Core Design Philosophy

**Think before coding.** Consider: purpose, tone, constraints, target audience, and what makes this design different from generic output.

> "Bold maximalism and refined minimalism both work - the key is intentionality, not intensity."

> "Claude is capable of extraordinary creative work. Don't hold back, show what can truly be created when thinking outside the box."

## Critical Implementation Areas

### Typography
- NEVER use generic fonts like Arial, Inter, or Roboto as primary display fonts
- Choose fonts that are beautiful, unique, and interesting
- Select characterful display and body font pairings that elevate aesthetics
- Use font pairings that create visual hierarchy and personality

### Color & Theme
- Commit to a cohesive aesthetic using CSS variables
- Avoid clichéd color schemes, especially purple gradients on white backgrounds
- Build a complete color system, not just a primary color

### Motion & Animation
- Use animations for high-impact moments and meaningful micro-interactions
- Prioritize one well-orchestrated page load with staggered reveals over scattered animations
- Animation should serve the design vision, not decorate it

### Spatial Composition
- Employ unexpected layouts: asymmetry, overlap, diagonal flow, grid-breaking elements
- White space is a design element, use it intentionally
- Break from predictable grid layouts when appropriate

### Visual Details
- Create atmosphere through gradients, textures, and decorative effects
- Every detail should reinforce the aesthetic direction
- Depth, shadow, and layering add sophistication

## What to AVOID

- Overused font families (Arial, Inter, Helvetica as display fonts)
- Purple gradients on white backgrounds (clichéd AI aesthetic)
- Predictable, cookie-cutter layouts lacking context-specific character
- Generic component libraries without customization
- Designs that could have been generated by any AI without thought

## Production Standards

Generated code must be:
- **Functional**: Everything works, no placeholder content
- **Visually striking**: Bold aesthetic choices executed with precision
- **Cohesive**: All elements reinforce the design direction
- **Complete**: Match implementation complexity to the aesthetic vision

Maximalist designs require elaborate code with extensive animations. Minimalist designs demand precision in spacing, typography, and subtle details.

## Workflow

1. **Analyze** the request: What is the purpose? Who is the audience? What tone?
2. **Define** a bold aesthetic direction before writing any code
3. **Select** typography and color system that embodies the direction
4. **Build** with intentionality — every choice should be deliberate
5. **Review** — does this look like distinctive work or generic AI output?
